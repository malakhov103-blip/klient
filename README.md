# Binio Client Bot

Это отдельный бот для клиентской анкеты. Он не связан с партнёрским ботом и должен запускаться отдельным сервисом на Railway.

## Файлы для GitHub

Загрузите только эти файлы:

1. client_bot_local.py
2. requirements.txt
3. railway.json
4. runtime.txt
5. README.md

Не загружайте:

- client_bot_data.pickle
- __pycache__
- .env

## Railway Variables

В Railway добавьте переменные именно у сервиса анкетного бота:

CLIENT_BOT_TOKEN=токен анкетного бота из BotFather
CLIENT_REQUESTS_CHAT_ID=-1003908716746

Если подключите Railway Volume, бот сам сохранит память в Volume через RAILWAY_VOLUME_MOUNT_PATH.

Дополнительно можно оставить стандартно:

MAX_TEXT_ANSWER_LENGTH=700

Это ограничивает слишком длинные ответы клиента, чтобы заявка не становилась огромной и бот не падал на отправке.

## Что нужно сделать в Telegram

1. Добавьте анкетного бота в чат заявок.
2. Дайте ему право писать сообщения.
3. Убедитесь, что CLIENT_REQUESTS_CHAT_ID указывает именно на этот чат.

Если бот не добавлен в чат заявок или у него нет права писать сообщения, анкета заполнится, но заявка не сможет отправиться. Пользователь увидит понятное предупреждение.

## Важно

Не запускайте один и тот же bot token в двух местах одновременно.

Партнёрский бот и анкетный бот должны быть двумя разными Railway-сервисами и иметь разные токены.

## Проверки

В этой версии:

- токен не хранится в коде
- старые кнопки анкеты не записывают ответ в неправильный вопрос
- если клиент отправил фото или файл вместо ответа, бот объясняет, что нужно сделать
- если клиент ввёл слишком длинный текст, бот аккуратно сокращает ответ
- если заявка слишком длинная, бот делит её на несколько сообщений в чате заявок
